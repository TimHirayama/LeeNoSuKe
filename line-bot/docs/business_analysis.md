# LINE Bot 商業營運與免費額度分析報告

以目前我們打造的「多輪狀態機 + RAG 發言人 + 翻譯機」公版架構，完全依賴免費方案能撐多久？這份報告為你解析四大資源瓶頸，以及未來為客戶量身打造 (B2B/B2C) 時的伺服器佈署策略。

## 1. 狸之助目前配置的極限分析 (皆為免費版)

### 🟢 LINE Messaging API (官方帳號)
- **回覆訊息 (Reply Message) 額度**：**無限且永遠免費！**
  只要使用者傳一句話過來，機器人在幾秒內呼叫 `client.replyMessage()` 回覆，不管對話幾萬次都不用錢。
- **推播訊息 (Push Message) 額度**：台灣輕量方案 **每月 200 則**。
  目前目前的公版架構並沒有使用到主動推播（全都是一問一答的回覆），因此 LINE 的部分絕對不會產生費用。

### 🟠 Gemini API (Google AI Studio)
這是目前最大的效能瓶頸（也是免費版的限制）：
- **頻率限制 (Rate Limit)**：**15 RPM** (每分鐘 15 次請求)。
  *如果在群組開啟「開始口譯」，且群組在 60 秒內丟出超過 15 句話，第 16 句話的翻譯就會拋出 `429 Too Many Requests` 錯誤。*
- **每日數量 (Daily Limit)**：**1,500 RPD** (每天 1,500 次呼叫)。
  *對於個人或單一群組來說，一天 1,500 句對話/翻譯非常非常夠用。*

### 🟡 Render Cloud (伺服器託管)
- **免費時數**：每個帳號每月 **750 小時**。
  *(一個月 31 天 * 24 小時 = 744 小時)。這剛好夠「一台」機器 24 小時不中斷運作（搭配 UptimeRobot 喚醒）。*
- **記憶體與 CPU**：512 MB RAM / 0.1 CPU，對稍微編譯過的 Node.js / Express 應用程式來說綽綽有餘。

### 🟢 Google Calendar API
- 每專案每天 **1,000,000 次** 查詢上限。這輩子對於這支專案來說基本上等於無限。

## 2. 商業化發布建議 (幫客戶做他們自己的機器人)

如果你要把這套「公版」複製 (Clone)，幫客戶 A 做「陳老闆特助」，幫客戶 B 做「林醫生發言人」，以下是免費與低成本營運的教戰守則：

### 方案 A：完全免費客家流 (適合極度低成本測試)
每個客戶都必須「**完全物理隔離**」：
1. **Gemini API**：請客戶用他們自己的 Google 帳號去申請一個免費的 `GEMINI_API_KEY`，確保這個客戶獨享每天 1500 次的對話量。
2. **Render 伺服器**：你必須幫每個客戶辦一個*全新的 Render 帳號*。因為一個 Render 帳號的 750 小時免費用量只夠支撐「一個群組/一隻機器人」，如果同帳號開兩隻，半個月後兩隻都會休眠鎖死。

### 方案 B：微型創業流 (月付一台 VPS ，接無數個客戶) 🔥最強推
Render 是非常棒的開發生產環境，但若你要「量產」客戶專案，**租一台虛擬主機 (VPS)** 是唯一的解法。
- **成本**：每月只要 **$5 美金** (約 150 台幣)。可以選擇 AWS Lightsail, DigitalOcean 或是 Linode。
- **作法**：
  這 $5 美金的主機大約有 1GB RAM，你可以利用 `PM2` 或是 `Docker` 在這台主機上同時啟動「客戶A、客戶B、客戶C... 等 10 隻到 20 隻以上的 Node.js Bot」。
- **利潤**：你每個月只需支出 $5 美金的電費，客戶只要自備他們的 LINE Token 和 Gemini 即可。對每個客戶你可以收取每月 $500 ~ $2,000 的「維護與技術授權費」，**完全是一本萬利！**

## 總結
這套由你親自催生的「RAG 發言人框架」技術難度適中、護城河卻很深（具備群組通訊與個人助理分離權限）。用上述的「微型創業流 VPS 方案」，加上每個客戶自己免費的 Gemini Key，你完全可以開始實現在接案市場的被動收入了！
